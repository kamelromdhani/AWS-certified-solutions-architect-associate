The Spread placement strategy strictly places a small group of instances across distinct underlying hardware to reduce correlated failures.

How does the Multi-Value routing policy work in Route 53?

The Multi-Value routing policy in Route 53 works as follows: - When a DNS query is received, Route 53 returns multiple values/resources in the DNS response, each with the same DNS name but different IP addresses. - Route 53 can associate health checks with the resources, and it only includes the IP addresses of the healthy resources in the response. - By specifying multiple values, the DNS resolver randomly selects one of the healthy IP addresses, distributing the traffic across the available resources. - Up to eight healthy records can be returned for each Multi-Value query. - Multi-Value routing is not a substitute for having an Elastic Load Balancer (ELB), but it can be used in conjunction with ELBs to provide a simple form of load balancing.

How does the Failover routing in Route 53 work?

The Failover routing in Route 53 is used for active-passive failover setups. It works as follows: - Two or three records are created with the same DNS name but different types: primary and secondary (and optionally, a tertiary record). - One record is designated as the primary and the others as failover resources. - Route 53 monitors the health of the primary resource using a health check. - When the primary resource is healthy, Route 53 responds to DNS queries with the IP address of the primary resource. - If the primary resource becomes unhealthy, Route 53 automatically switches the DNS response to one of the failover resources, directing traffic to the backup resource. - Once the primary resource becomes healthy again, Route 53 switches back to routing traffic to the primary resource.

What is a Bastion Host and how is it used in AWS?

A Bastion Host is a special EC2 instance that is used as a jump host for SSH access to private EC2 instances in a VPC. It is typically placed in a public subnet and its security group must allow inbound traffic on port 22 from restricted CIDR. The security group of the EC2 instances must allow traffic from the security group of the Bastion Host, or the private IP of the Bastion Host.

What is a Network Access Control List (NACL) in AWS and how does it work?

A Network Access Control List (NACL) in AWS is like a firewall that controls traffic from and to subnets. Each subnet has its own NACL, and newly created subnets are assigned the default NACL. NACL rules have a number, and the first rule match drives the decision. The last rule is an asterisk (*) and denies a request in case of no rule match. AWS recommends adding rules by increment of 100. Newly created NACLs will deny everything, and NACLs are a great way of blocking a specific IP address at the subnet level.

What are the types of VPC endpoints in AWS?

There are two types of endpoints in AWS:
  • Interface endpoints and gateway endpoints. Interface endpoints are powered by PrivateLink and provision an ENI (private IP address) as an entry point. They support most AWS services and have a cost per hour plus a cost per GB of data processed.
  • Gateway endpoints, on the other hand, provision a gateway and must be used as a target in a route table. They do not use security groups and support both S3 and DynamoDB. Gateway endpoints are free.

What is AWS VPN CloudHub?

AWS VPN CloudHub is a service provided by AWS that allows you to establish secure connections between multiple VPCs over the internet. It provides a hub-and-spoke model for connecting VPCs, with the hub being the central point of communication and the spokes being the individual VPCs. AWS VPN CloudHub simplifies the process of connecting multiple VPCs by providing a single VPN connection for all VPCs.

What is an egress-only Internet Gateway?

An egress-only internet gateway is a horizontally scaled, redundant, and highly available VPC component that allows outbound communication over IPv6 from instances in your VPC to the internet and prevents the internet from initiating an IPv6 connection with your instances.

What is AWS Network Firewall?

AWS Network Firewall is a service that allows you to protect your entire Amazon VPC. It can inspect VPC to VPC traffic, outbound to the internet, inbound from the internet, and to/from Direct Connect & Site-to-Site VPN. The AWS Network Firewall uses the AWS Gateway Load Balancer internally. Rules can be centrally managed cross-account by AWS Firewall Manager to apply to many VPCs.

What does RTO mean in disaster recovery solutions?

Recovery Time Objective (RTO) is the duration of time and a service level within which a business process must be restored after a disaster in order to avoid unacceptable consequences associated with a break in continuity. In other words, the RTO is the answer to the question: How much time did it take to recover after notification of business process disruption?

What does RPO mean in disaster recovery solutions?

RPO is your goal for the maximum amount of data the organization can tolerate losing. This parameter is measured in time: from the moment a failure occurs to your last valid data backup. For example, if you experience a failure now and your last full data backup was 24 hours ago, the RPO is 24 hours. 

Explain the Pilot Light disaster recovery strategy?

  • A small version of the app is always running in the cloud
  • Useful for the critical core (pilot light)
  • Very similar to Backup and Restore
  • Faster than Backup and Restore as critical systems are already up

Explain the disaster recovery strategy Warm Standby

  • Full system is up and running, but at minimum size
  • Upon disaster, we can scale to production load

Explain the disaster recovery strategy Hot Site / Multi Site Approach

  • Very low RTO (minutes or seconds) very expensive
  • Full Production Scale is running AWS and On Premise

What is AWS Server Migration Service (SMS)?

AWS Server Migration Service (SMS) is a service that provides incremental replication of on-premise live servers to AWS.

What is AWS Application Migration Service (MGN)?

  • AWS Application Migration Service is the "AWS evolution" of CloudEndure Migration, and it replaces AWS Server Migration Service (SMS).
  • It is a lift-and-shift (rehost) solution that simplifies migrating applications to AWS.
  • MGN can convert physical, virtual, and cloud-based servers to run natively on AWS.
  • It supports a wide range of platforms, operating systems, and databases.
  • MGN promises minimal downtime and reduced costs during migration.

What's ENA (Elastic Network Adapter)?

ENA (Elastic Network Adapter) is to enhance your EC2 networking and gives you higher bandwidth, higher package per second, and lower latency.

What's Elastic Fabric Adapter (EFA)

  • Improved ENA for HPC (High Performance Computing), only works for Linux
  • Great for inter-node communications, tightly coupled workloads
  • Leverages Message Passing Interface (MPI) standard
  • Bypasses the underlying Linux OS to provide low-latency, reliable transport

What's AWS ParallelCluster?

AWS ParallelCluster is an open source cluster management tool that makes it easy for you to deploy and manage High Performance Computing (HPC) clusters on AWS. ParallelCluster uses a simple graphical user interface (GUI) or text file to model and provision the resources needed for your HPC applications in an automated and secure manner. It also supports multiple instance types and job submission queues, and job schedulers like AWS Batch and Slurm. You can enable EFA on the cluster to improve network performance.

What are the differences between AWS Batch and AWS Lambda?

AWS Lambda has a time limit, limited runtimes, limited temporary disk space, and is serverless, whereas AWS Batch has no time limit, any runtime as long as it's packaged as a Docker image, relies on EBS/instance store for disk space, and relies on EC2 (can be managed by AWS).

What is Amazon AppFlow?

Amazon AppFlow is a fully managed integration service that enables secure data transfer between Software-as-a-Service (SaaS) applications and AWS. It supports sources such as Salesforce, SAR, Zendesk, Slack, and ServiceNow, and destinations like AWS services such as Amazon S3, Amazon Redshift, and non-AWS destinations such as SnowFlake and Salesforce.
